# Signal Scout References

This folder contains supporting material that is useful context but should not drive active implementation unless explicitly relevant.

## Folders

- `auctionsplus/` - AuctionsPlus assessment forms, manuals, and reference cards.
- `agents/` - cattle agent website, newsletter, and PDF source research.
- `operations/` - security, IP, and interface-planning references.
- `platforms/` - Australian cattle listing platform research.
- `signal-model/` - deferred Signal Model / cattle market prediction material.

## Notes

- Files ending in `.ap-export.zip` are Claude/knowledge-library exports that contain page images, extracted text, and a manifest. They were renamed from misleading `.pdf` filenames because they are not true PDFs.
- `references/operations/source-register.md` is the working source inventory for expansion decisions.
- `references/operations/permission-outreach-list.md` is the working permission / partnership contact queue.
- `references/operations/outreach-log.md` is the send/follow-up record once source outreach begins.
- `references/operations/permission-email-drafts.md` contains current outreach drafts and verified contact paths.
- `references/operations/authorised-intake-ops.md` is the practical mailbox, shortlist, and raw-notice intake checklist.
- `references/operations/kill-the-newsletter-intake-workflow.md` explains when Kill the Newsletter is an acceptable authorised-email transport path for Signal Scout and how to operationalise it safely.
- `references/operations/authorized_subscription_register_template.csv` is the starter register for authorised subscriptions.
- `references/operations/authorized_notice_batch_template.json` is the raw notice batch template for manual exports.
- `references/operations/firecrawl-allowed-use-checklist.md` defines when Firecrawl is a valid technical extractor for Signal Scout and where it does not change the permission posture.
- `references/operations/source-integration-architecture.md` is the implementation-facing source-family and rollout architecture for multi-source expansion.
- `references/operations/parser-feasibility-shortlist.md` ranks parser-first sources by extractability and build value.
- `references/operations/parser-feasibility-field-map.md` maps likely field coverage across parser-first sources.
- `references/operations/parser-specs-top-three.md` defines first-draft parser specs for Stockplace, Sullivan, and RMA.
- `references/operations/ai-content-pipeline-research-review-for-signal-scout.md` distils reusable scraping and ingestion lessons from an external AI pipeline research note into Signal Scout-specific guidance.
- Signal Model references are intentionally separated from active Signal Scout beta docs.
- Do not place secrets, live spreadsheets, or raw credential files in this folder.
