# Workspace Setup Questionnaire

Use this when adapting the same ICM scaffold to another project.

1. What are the stable product goals and the current top priority?
2. Which files are the live runtime entry points?
3. Which docs are the active source of truth?
4. Which folders are historical or background only?
5. What validations prove a change is safe?
6. Which files must never move casually because scripts, operators, or tooling depend on them?
7. What recurring task types deserve their own stage contracts?
