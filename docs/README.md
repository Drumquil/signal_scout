# Signal Scout Docs

This folder contains the active Codex-facing project documentation for Signal Scout.

## Files

- `CODEX_BRIEF.md` - compact product and technical snapshot for fast onboarding.
- `PROJECT_MAP.md` - where to start by task type and file path.
- `CURRENT_STATE.md` - active status, blockers, and next actions.
- `beta-sequence.md` - concrete order for beta testing, tightening, and later source outreach.
- `beta-user-guide.md` - short guide to send to first beta testers.
- `beta-feedback-process.md` - minimal feedback loop for early real-user testing.
- `status.md` - compact detailed state and locked decisions.
- `schema.md` - current Google Sheets schema and tab contracts.
- `auctionsplus-selector-reference.md` - current AuctionsPlus extraction reference.
- `prelaunch-checklist.md` - beta, launch, legal, technical, and operational gates.
- `strategy.md` - product strategy, market position, and architecture direction.

## Rules

- Start with `CODEX_BRIEF.md`, `PROJECT_MAP.md`, and `CURRENT_STATE.md`.
- Read deeper docs only when the task touches that area.
- Keep secrets, live credentials, raw `.env` values, and private phone numbers out of docs.
- Move historical or third-party material to `references/` rather than keeping it in this active docs folder.
- Use `../archive/` for historical handovers, superseded schema references, and older state documents such as `status-history.md`.
